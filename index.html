Пример

Как выглядит сгенерированный spec

Ввели: «добавить сброс пароля по email» — получили:

── Product Spec ──

Title: "Сброс пароля по email"
Goal: "Дать пользователю восстановить доступ через email"
Priority: High
Scope:
├─ FE: форма "Забыли пароль?" + форма сброса с токеном
├─ BE: POST /auth/reset-password, POST /auth/reset-password/:token
└─ Email: шаблон письма с одноразовой ссылкой (TTL 30 мин)

── Dev Checklist ──
☐ Создать таблицу reset_tokens в БД
☐ Написать email-сервис (SendGrid / SES)
☐ Реализовать endpoint генерации токена
☐ Реализовать endpoint верификации токена
☐ Сверстать форму сброса (мобильная + десктоп)
☐ Rate limiting: не чаще 1 запроса / 60 с

── Acceptance Criteria ──
✅ Пользователь получает письмо за <5 с
✅ Ссылка работает только 1 раз
✅ После сброса старый пароль недействителен
✅ Ошибка: неверный/просроченный токен → понятное сообщение
// 3 test scenarios generated automatically
